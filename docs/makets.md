# Интерфейс и экранные формы

### Окно авторизации

(Ссылка на изображение: *Рисунок – окно авторизации*)
[окно-авторизации](./makets/окно-авторизации.png)

В окне авторизации необходимо указывать логин сотрудника и пароль. Для пароля необходимо реализовать следующие ограничения:
• Пароль должен быть не менее 8 символов;
• В пароле должны присутствовать буквы верхнего и нижнего регистров, цифры и специальные символы (например, !, @, #, $, %, &, *, +, -, =, ., / и др.);
• Пароль не должен содержать последовательности символов (qwerty, 12345 и т. п.);
• Пароль не должен полностью совпадать с логином.
Пароль должен быть скрыт в поле с вводом, однако необходимо реализовать кнопку, при нажатии на которую можно его посмотреть.
Необходимо реализовать уведомление для пользователей, если ввод логина и пароля был выполнен ошибочно.

### Экранная форма стартовой страницы

(Ссылка на изображение: *Рисунок – экранная форма стартовой страницы*)
[экранная-форма-стартовой-страницы](./makets/экранная-форма-стартовой-страницы.png)

На всех страницах в системе необходимо реализовать следующий функционал:
• Кнопка с уведомлениями, при нажатии на которую будет отображаться информация по изменениям в заявках;
• Профиль текущего пользователя системы;
• Кнопка выхода из системы.

На стартовой странице должны быть разработаны следующие разделы:
• **“Создание заявки”** – при нажатии открывается форма для создания заявки;
• **“Просмотр заявок”** – переход на страницу для просмотра и взаимодействия с заявками. Сотрудник с ролью “Автор” может взаимодействовать только с созданными им заявками; “Исполнитель” – только с назначенными на него и созданными им; “Руководитель” – видит созданные для его отдела, делегированные в его отдел, а также созданные им заявки;
• **“Отчетность”** – доступен только для руководителей. Открывает окно для формирования отчетности со статистикой по выполнению заявок подчиненными;
• **“Управление сотрудниками”** – доступен только для руководителей. Переход на страницу для заведения/удаления сотрудников отдела, изменения признака активности;
• **“Виды работы”** – доступен только для руководителей. Страница для просмотра видов работ по отделу, создания новых и удаления текущих;
• **“Изменение приоритетности заявки”** – доступен только для руководителей. Раздел для изменения влияния величин при определении приоритета заявки.

### Экранная форма страницы для просмотра и работы с заявками

(Ссылка на изображение: *Рисунок – страница для просмотра и работы с заявками*)
[Экранная-форма-страница-для-просмотра-и-работы-с-заявками](./makets/Экранная-форма-страница-для-просмотра-и-работы-с-заявками.png)


На странице нужно реализовать список заявок в зависимости от роли:
• **Автор** – только собственные заявки;
• **Исполнитель** – заявки, на которые он назначен (или был назначен), закрытые им, а также созданные им как автором;
• **Руководитель** – заявки, назначенные и делегированные на его отдел, а также созданные им как автором.

Сортировка для всех (выпадающий список): по приоритету, статусу, дате создания, дате закрытия.
Фильтрация:
• Поиск по ФИО исполнителя (строка ввода, Enter)
• Поиск по номеру заявки (строка ввода, Enter)
• “Созданные мной” (для исполнителя и руководителя)
• “Делегированы из другого отдела” (только для руководителя)
• “Назначенные на меня” (только для исполнителя)

При выборе заявки из списка она отображается справа для взаимодействия. Должна быть возможность скрыть список, сдвинув его влево.

### Экранная форма окна для создания заявки

(Ссылка на изображение: *Рисунок – окно для создания заявки*)
[Экранная-форма-окна-для-создания-заявки](./makets/экранная-форма-окна-для-создания-заявки.png)


**Описание макета:**
• Поле “Тема” (краткое описание проблемы)
• Поле “Департамент” (выбор подразделения)
• Поле “Вид работ” (зависит от департамента)
• Поле с описанием проблемы (возможность скроллинга, максимум 1000 символов)
• Выбор шрифта для текста описания
• Возможность прикреплять файлы (фото)
• Кнопка “Отправить” – при незаполненных полях – уведомление и подсветка незаполненных полей.
