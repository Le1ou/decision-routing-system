## Статусная модель заявки

> Таблица приведена в соответствие с фактическим конечным автоматом backend
> (доступные действия и их обработка). Перечень внесённых правок — в
> `docs/doc-updates.md`.

(Ссылка на изображение: *Рисунок – статусная модель заявки*)
[статусная-модель](./makets/статусная-модель.png)

**Статусы заявки** (в скобках — внутренний идентификатор):
• Новый (`new`)
• Назначен исполнитель (`assigned`)
• Делегирована в другой отдел (`delegated`)
• В работе (`inProgress`)
• Отклонена (`rejected`)
• Завершена (`completed`)

Дополнительно у заявки есть признак **архивирования** (`archivedAt`) — это не статус:
завершённые и отклонённые заявки можно убрать из основного списка; отклонённые
скрываются автоматически через 7 дней. Действие в скобках в таблице — это `action`
из `POST /applications/{id}/actions`; набор доступных действий backend возвращает в
карточке заявки (`availableActions`).

**Таблица изменения статусов:**

| Текущий статус | Целевой статус | Описание |
|----------------|----------------|----------|
| - | Новый | Заявка создана автором. Исполнитель назначается автоматически подсистемой маршрутизации или вручную руководителем. |
| Новый | Назначен исполнитель | Руководитель назначил исполнителя (`assignExecutor`), либо автоматическое назначение подсистемой маршрутизации: свободный минимально способный исполнитель отдела по матрице грейдов, очередь по убыванию приоритета (критичная заявка распределяется немедленно при создании). |
| Новый | Делегирована в другой отдел | Руководитель делегировал заявку в другой отдел (`delegateExternal`). |
| Новый | Отклонена | Заявка отменена автором или руководителем (`cancel`). |
| Назначен исполнитель | В работе | Исполнитель приступил к работе (`startWork`). |
| Назначен исполнитель | Отклонена | Заявка отклонена руководителем (`reject`). |
| Назначен исполнитель | Делегирована в другой отдел | Делегирование в другой отдел руководителем или назначенным исполнителем (`delegateExternal`). |
| Назначен исполнитель | Новый | Возврат для перераспределения: руководитель (`returnToNew`), «отказ» исполнителя (`reject`) или делегирование в рамках отдела без подтверждения (`delegateInternal`). Признак «Незавершённая», сохраняется предыдущий исполнитель. |
| Назначен исполнитель | Делегирована в другой отдел | Делегирование в рамках своего отдела, если у отдела включено подтверждение руководителя (`delegateInternal` при `delegated_to_same_dep`). |
| В работе | Завершена | Исполнитель закрыл заявку с результатом работ (`complete`). |
| В работе | Отклонена | Заявка отклонена руководителем (`reject`). |
| В работе | Новый | Возврат для перераспределения: руководитель (`returnToNew`), «отказ» исполнителя (`reject`) или делегирование в рамках отдела без подтверждения (`delegateInternal`). Признак «Незавершённая». |
| В работе | Делегирована в другой отдел | Делегирование в рамках своего отдела с подтверждением руководителя (`delegateInternal` при `delegated_to_same_dep`). |
| Делегирована в другой отдел | Новый | Подтверждение внешнего делегирования (`confirmExternalDelegation`) — заявка переходит во владение целевого отдела; либо отклонение (`declineExternalDelegation`) — заявка остаётся в исходном отделе. |
| Делегирована в другой отдел | Назначен исполнитель | Руководитель назначил исполнителя напрямую (`assignExecutor`, отменяет делегирование); либо отклонение делегирования в рамках отдела, если работа ещё не начиналась. |
| Делегирована в другой отдел | В работе | Отклонение делегирования в рамках отдела, если работа уже шла (восстановление прежнего статуса). |
| Назначен исполнитель / В работе | Новый | Вытеснение по правилу «одна заявка на исполнителя»: при назначении исполнителя на другую заявку его прежняя активная заявка освобождается (признак «Незавершённая», сохраняется предыдущий исполнитель). |
| Назначен исполнитель / В работе | Новый | Вытеснение критичной заявкой (маршрутизация): если свободных подходящих исполнителей нет, критичная заявка вытесняет наименее приоритетную НЕ критичную заявку подходящего исполнителя; вытесненная возвращается в «Новый» (признак «Незавершённая», сохраняется предыдущий исполнитель). Критичные текущие заявки не вытесняются; если вытеснять некого — критичная остаётся в «Новый», руководителю уходит уведомление-эскалация. |
| Завершена | - | Заявка выполнена. Может быть заархивирована (`archive`). |
| Отклонена | - | Заявка отклонена. Может быть заархивирована (`archive`); скрывается из списка автоматически через 7 дней. |
