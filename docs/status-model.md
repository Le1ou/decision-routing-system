# Статусная модель заявки

Статусная модель описывает конечный автомат заявки и действия, которые переводят заявку
между состояниями. Архивирование не является статусом: оно только скрывает завершенную
или отклоненную заявку из основного списка.

[статусная-модель](./makets/статусная-модель.png)

## Статусы

| Статус | Идентификатор | Смысл |
|--------|---------------|-------|
| Новый | `new` | Заявка создана, ожидает автоматического распределения или ручного назначения. |
| Назначен исполнитель | `assigned` | Исполнитель выбран, но работа еще не начата. |
| Делегирована | `delegated` | Заявка ожидает решения руководителя по внешнему или внутреннему делегированию. |
| В работе | `inProgress` | Исполнитель приступил к выполнению заявки. |
| Отклонена | `rejected` | Заявка закрыта без выполнения или отменена в статусе `new`. |
| Завершена | `completed` | Исполнитель завершил работы и указал результат. |

## Переходы

| Откуда | Куда | Действие | Инициатор | Условия и результат |
|--------|------|----------|-----------|---------------------|
| - | `new` | создание | Автор | Создается заявка с описанием, отделом, видом работ и дедлайном. Backend рассчитывает приоритет и записывает начальное событие в историю. |
| `new` | `assigned` | `assignExecutor` или автоматическая маршрутизация | Руководитель / система | Назначается исполнитель. При автоматическом назначении проверяются отдел, активность, роль исполнителя, грейд, должность, занятость и кулдаун. |
| `new` | `delegated` | `delegateExternal` | Руководитель | Заявка отправляется на подтверждение руководителю другого отдела. |
| `new` | `rejected` | `cancel` | Автор / руководитель | Новая заявка отменяется. Заполняются дата закрытия и пользователь, закрывший заявку. |
| `new` | `new` | фоновая маршрутизация без назначения | Система | Если нет свободного подходящего исполнителя, заявка остается в очереди; руководителю отправляется дедуплицированное уведомление. |
| `assigned` | `inProgress` | `startWork` | Назначенный исполнитель | Исполнитель берет заявку в работу, фиксируется дата начала работы. |
| `assigned` | `assigned` | `assignExecutor` | Руководитель | Руководитель меняет исполнителя. Предыдущая связь исполнителя заменяется. |
| `assigned` | `delegated` | `delegateExternal` | Руководитель / назначенный исполнитель | Заявка отправляется в другой отдел на подтверждение. |
| `assigned` | `delegated` | `delegateInternal` | Назначенный исполнитель | Внутреннее делегирование требует подтверждения руководителя отдела. Исполнитель задает сложность не ниже текущей. |
| `assigned` | `new` | `delegateInternal` | Назначенный исполнитель | Если подтверждение внутри отдела не требуется, заявка возвращается в очередь как незавершенная. |
| `assigned` | `new` | `reject` | Назначенный исполнитель | Для исполнителя отказ от заявки означает возврат в `new` с признаком незавершенности. |
| `assigned` | `new` | `returnToNew` | Руководитель | Руководитель возвращает заявку для перераспределения, сохраняется предыдущий исполнитель. |
| `assigned` | `rejected` | `reject` | Руководитель | Руководитель отклоняет заявку и закрывает ее без выполнения. |
| `inProgress` | `completed` | `complete` | Назначенный исполнитель | Исполнитель завершает заявку, указывает результат работ, фиксируется дата завершения. |
| `inProgress` | `assigned` | `assignExecutor` | Руководитель | Руководитель переназначает заявку; статус остается назначенным новому исполнителю, дата назначения обновляется. |
| `inProgress` | `delegated` | `delegateInternal` | Назначенный исполнитель | Внутреннее делегирование требует подтверждения руководителя; заявка ждет решения. |
| `inProgress` | `new` | `delegateInternal` | Назначенный исполнитель | Если подтверждение внутри отдела не требуется, заявка возвращается в очередь как незавершенная. |
| `inProgress` | `new` | `reject` | Назначенный исполнитель | Для исполнителя отказ от заявки означает возврат в `new` с признаком незавершенности. |
| `inProgress` | `new` | `returnToNew` | Руководитель | Руководитель возвращает заявку для перераспределения, сохраняется предыдущий исполнитель. |
| `inProgress` | `rejected` | `reject` | Руководитель | Руководитель отклоняет заявку и закрывает ее без выполнения. |
| `delegated` | `new` | `confirmExternalDelegation` | Руководитель принимающего отдела / руководитель отдела | При внешнем делегировании заявка меняет отдел на принимающий. При внутреннем делегировании заявка возвращается в очередь своего отдела как незавершенная. |
| `delegated` | `new` | `declineExternalDelegation` | Руководитель принимающего отдела | Внешнее делегирование отклонено; заявка остается в исходном отделе и возвращается в очередь. |
| `delegated` | `assigned` | `declineExternalDelegation` | Руководитель отдела | Внутреннее делегирование отклонено до начала работ; заявка возвращается прежнему исполнителю в `assigned`. |
| `delegated` | `inProgress` | `declineExternalDelegation` | Руководитель отдела | Внутреннее делегирование отклонено после начала работ; заявка возвращается прежнему исполнителю в `inProgress`. |
| `delegated` | `assigned` | `assignExecutor` | Руководитель | Руководитель назначает исполнителя напрямую, активное делегирование закрывается как отклоненное. |
| `assigned` / `inProgress` | `new` | вытеснение при назначении | Руководитель / система | Если исполнителя назначили на другую заявку, его прежние активные заявки возвращаются в `new` как незавершенные. |
| `assigned` / `inProgress` | `new` | вытеснение критичной заявкой | Система | Критичная заявка может вытеснить активную некритичную заявку подходящего исполнителя. Вытесненная заявка возвращается в очередь. |
| `completed` | `completed` | `archive` | Руководитель в зоне видимости | Заявка получает `archivedAt` и скрывается из основного списка. Статус не меняется. |
| `rejected` | `rejected` | `archive` | Руководитель в зоне видимости | Заявка получает `archivedAt` и скрывается из основного списка. Отклоненные заявки также автоматически скрываются из списка через настроенный срок. |

## Дополнительные правила

- Любая смена статуса записывается в `application_status_history`.
- `availableActions` рассчитываются backend с учетом статуса, роли, авторства,
  назначения исполнителем и зоны ответственности руководителя.
- В статусах `completed` и `rejected` бизнес-действия над заявкой завершены; чат
  доступен только на чтение.
- Заявка с `archivedAt` исключается из основного списка, но доступна в данных и
  сохраняет исходный статус.
- Отклоненные заявки без ручного архивирования скрываются из основного списка через
  срок, заданный настройкой backend `applications.rejected_visible_days`.
