# Виды работ и матрицы допуска

Документ описывает справочник видов работ, связь видов работ с отделами, сложностью,
грейдами и должностями, а также правила использования этих данных при маршрутизации.

## 1. Назначение справочника

1.1. Вид работ определяет тип производственной задачи, которую автор выбирает при
создании заявки.

1.2. Вид работ принадлежит конкретному отделу. При создании заявки автор выбирает
отдел и вид работ этого отдела.

1.3. Вид работ используется:
- для классификации заявок;
- для определения базовой сложности;
- для автоматического подбора исполнителя;
- для отчетности и аналитики по типам работ.

## 2. Атрибуты вида работ

2.1. Вид работ содержит:
- идентификатор;
- название;
- отдел;
- базовую сложность;
- список допустимых грейдов;
- список допустимых должностей.

2.2. Базовая сложность принимает значения `easy`, `medium`, `hard`.

2.3. Список допустимых грейдов обязателен. Вид работ без допустимых грейдов не может
быть корректно использован автоматической маршрутизацией.

2.4. Список допустимых должностей необязателен. Пустой список означает отсутствие
ограничения по должности: подходит любая должность с допустимым грейдом.

## 3. Должности и грейды

3.1. Должность сотрудника приходит из AD и соответствует справочнику `post`.

3.2. Грейд сотрудника определяется через связку должности и грейда в `post_grade`.

3.3. Должность и грейд являются двумя независимыми осями допуска к виду работ:
исполнитель подходит виду работ только если проходит обе проверки.

3.4. Должность не участвует в формуле приоритета заявки. Она используется только для
допуска исполнителя при автоматической маршрутизации.

## 4. Управление видами работ

4.1. Все авторизованные пользователи могут получать список видов работ и фильтровать
его по отделу.

4.2. Руководитель может создавать, изменять и удалять виды работ только своего отдела.

4.3. Топ-менеджер может управлять видами работ любого отдела.

4.4. При создании или изменении вида работ руководитель задает название, отдел,
сложность, допустимые грейды и при необходимости допустимые должности.

4.5. Удаление вида работ запрещено, если на него ссылается хотя бы одна заявка. В этом
случае backend возвращает конфликт.

4.6. При изменении списка грейдов или должностей матрица допуска заменяется целиком.

## 5. Использование в заявке

5.1. При создании заявки выбранный вид работ записывается в карточку заявки.

5.2. Если исполнитель инициирует внутреннее делегирование, он может изменить вид работ
и обязан указать сложность не ниже текущей.

5.3. Руководитель может изменить вид работ заявки в статусе `new`.

5.4. Назначенная исполнителем сложность хранится отдельно от базовой сложности вида
работ. Если назначенная сложность отсутствует, интерфейс может использовать сложность
вида работ как базовую.

## 6. Использование в автоматической маршрутизации

6.1. Автоматическая маршрутизация учитывает вид работ при подборе свободного
исполнителя.

6.2. Исполнитель считается подходящим по виду работ, если:
- грейд исполнителя входит в `allowedGradeIds`;
- должность исполнителя входит в `allowedPositionIds` или `allowedPositionIds` пуст;
- сотрудник активен и находится в отделе заявки.

6.3. При наличии нескольких подходящих свободных исполнителей система выбирает
минимально способного: с наименьшим подходящим грейдом, затем дольше всех простаивавшего.

6.4. Ручное назначение руководителем не проверяет матрицы допуска. Это позволяет
руководителю принять исключительное управленческое решение.

## 7. Наполнение по умолчанию

Базовый seed содержит следующие отделы и виды работ. После запуска системы руководители
могут изменять справочник в рамках своих прав.

### IT-отдел

- Замена оборудования
- Починка оборудования
- Настройка сервера
- Установка ПО
- Настройка ПО

### Отдел главного энергетика

- Ремонт электродвигателей
- Устранение замыканий
- Ремонт проводки
- Ремонт трубопроводов
- Устранение обесточивания

### Производственный отдел

- Заявка на ремонт оборудования
- Заявка на замену оборудования

### Отдел контроля качества

- Проверка/приемка покупных деталей
- Проверка/приемка готовой продукции
- Фиксация брака
- Устранение брака

### Отдел главного механика

- Ремонтные работы
- Замена комплектующих
- Замена масла
- Аварийные работы

### Складской отдел

- Оформление товарных накладных
- Отгрузка товара
- Инвентаризация
- Приемка товара
- Списание и утилизация
- Приемка от производства внутри организации

### Отдел снабжения

- Предоставление комплектующих/деталей
- Оформление заказа поставщику
- Создание заявок на тендер
- Подготовка договоров
- Замена позиции в поставке
